# Open-science release checklist

Before making the repository public or archiving it with Zenodo, check:

- [ ] No API keys, passwords, private tokens, or local credentials are present.
- [ ] Raw data are excluded and data download instructions are documented.
- [ ] The license is correct for the intended release.
- [ ] `CITATION.cff` contains the final author list and repository URL.
- [ ] The manuscript citation or preprint DOI is added when available.
- [ ] The release tag is created, for example `v1.0.0`.
- [ ] The Zenodo DOI is added back to the README after archiving.
