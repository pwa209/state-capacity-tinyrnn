# Codex Manual: Mandatory Full-Research Expansion From the Current State-Capacity TinyRNN Implementation

## Project title

Machine-defined multidimensional state and capacity profiles in human cognition using compact recurrent neural networks

## Mission

Revise and extend the existing TinyRNN state-capacity implementation into a full, reproducible empirical study suitable for a serious Nature or Nature Human Behaviour submission. The study must implement the theoretical framework in which artificial neural networks are used as perturbable instruments to distinguish global computational state from structural capacity. Both constructs are treated as multidimensional profiles: state is an operating-regime profile, and capacity is a resource/geometry profile. Summary coordinates may be reported for visualization, but confirmatory claims must be based on profile convergence and robustness, not only on a single scalar. The final output must be a complete research package, not a pilot.

## Absolute rules

1. Do not run pilot-only analyses as final analyses.
2. Do not use manually downloaded datasets in the main paper.
3. Do not use any dataset requiring login, private approval, controlled access or browser-only download.
4. Do not report any empirical claim unless it appears in `claim_audit.tsv` with a script, output file, sample size and claim-strength label.
5. Do not hide negative validation results.
6. Do not claim predictive superiority unless the model beats deployable baselines on held-out participants.
7. Do not call fixed points biological attractors.
8. Do not claim consciousness diagnosis.
9. Do not proceed to human projection unless the artificial-agent intervention gate passes, unless the project is explicitly reframed as exploratory or negative.
10. Do not use `--sample`, `--max-subjects`, `head()`, `nrows`, debug subsets or early-break logic in full-run mode.

## Step 05: Machine perturbation library

Train baseline TinyRNN/GRU agents and generate perturbation families.

State family: fixed weights and architecture; vary these parameters:

```text
tau = [0.7, 1.0, 1.3, 1.6, 2.0]
lapse = [0.00, 0.02, 0.05, 0.10, 0.20]
sigma = [0.00, 0.01, 0.03, 0.05, 0.10]
update_gain = [1.00, 0.97, 0.93, 0.88, 0.80]
memory_decay = [0.00, 0.05, 0.10, 0.20]
```

Capacity family: fixed operating regime; vary these resources:

```text
hidden_size = [1, 2, 3, 4, 6, 8]
recurrent_rank = [1, 2, 4, full]
memory_window = [1, 3, 5, 10, full]
bottleneck_width = [1, 2, 4, full]
recurrence = [GRU, vanilla_RNN, no_recurrence]
```

Hybrid family: factorial crossing of state severity and capacity level.

Required outputs:

```text

outputs/tables/artificial_behavioral_fingerprints.csv
outputs/tables/artificial_dynamics_fingerprints.csv

Acceptance: each artificial agent must have a unique ID, perturbation family, seed, task set, parameter values and checkpoint path.

## Step 07: ANN-only intervention gate

Required analyses:

1. Classify state vs capacity perturbation family under performance matching.
2. Recover known state perturbation dimensions and their composite severity.
3. Recover known capacity-resource dimensions and their composite level.
4. Recover both multidimensional profiles in hybrid agents.
5. Repeat after residualizing mean accuracy and negative log-likelihood.
6. Repeat with shuffled perturbation labels.

```text

Capacity recovery was near-threshold or partial. Therefore, downstream human state claims must remain exploratory unless later profile projection supplies unusually strong convergent evidence; capacity claims must be framed as multi-axis proxy evidence until geometry and robustness steps converge.
